# Core-Auth Feature Index

This document outlines all the features available in the `core-auth` system. The functionality is organized by bounded contexts (domains), with each endpoint representing a distinct feature flow that can be individually documented in the implementation guide.

## 1. Authentication (`auth` domain)
- **`Register`**: Creates a new user account with an email and password, securely hashing the password (e.g., using bcrypt/Argon2) before storing it in the database.
- **`Login`**: Verifies user credentials against the stored hash, applies security policies (like tenant IP allow/denylists and brute-force protection), and issues access & refresh tokens.
- **`ForgotPassword`**: Initiates a password reset flow by generating a secure reset token and dispatching a recovery email or notification.
- **`ResetPassword`**: Consumes the reset token and updates the user's password securely.

## 2. Session Management (`session` domain)
- **`RefreshToken`**: Exchanges a valid refresh token for a new pair of access and refresh tokens, handling token rotation and invalidating the old token.
- **`ListSessions`**: Retrieves a list of all active sessions (e.g., active devices/browsers) for the authenticated user.
- **`RevokeSession`**: Terminates a specific target session by its ID, effectively logging the user out from that specific device.
- **`RevokeAllSessions`**: Terminates all active sessions for the user globally.
- **`Logout`**: Invalidates the currently active session used to make the request.

## 3. Multi-Factor Authentication (`mfa` domain)
- **`SetupTOTP`**: Begins TOTP setup by generating a cryptographic secret and provisioning an `otpauth://` URI and QR code for an authenticator app.
- **`ConfirmTOTP`**: Finalizes TOTP setup by verifying the first Time-based One-Time Password generated by the authenticator app, activating MFA.
- **`ChallengeMFA`**: Validates a TOTP code during the login flow to complete authentication when an account has MFA enforced.
- **`DisableMFA`**: Securely removes the TOTP secret and disables the MFA requirement for the user.

## 4. Social Login / OAuth (`oauth` domain)
- **`GetOAuthURL`**: Generates a secure authorization redirect URL infused with `state` and `PKCE` parameters for providers like Google or Apple.
- **`OAuthCallback`**: Handles the callback from a provider, exchanges the authorization code for metadata, and either provisions a new user or authenticates an existing one.
- **`LinkProvider`**: Attaches an external OAuth identity to a currently authenticated user account.
- **`UnlinkProvider`**: Detaches a configured OAuth provider identity from the user account.

## 5. User Profile (`user` domain)
- **`GetMe`**: Fetches the profile information, status, and metadata for the currently authenticated user.
- **`ChangePassword`**: Safely updates the user's password by verifying their old password first.

## 6. Administration (`admin` domain)
- **`SuspendUser`**: Blocks a user from accessing the system moving forward and immediately revokes all of their active sessions.
- **`UnsuspendUser`**: Restores system access for a previously suspended user.
- **`DeleteUser`**: Removes a user account completely (or handles soft-deletion logic), cascading cleanup to their sessions, MFA secrets, and OAuth links.

## 7. Passwordless & Verification (`verification` domain)
- **`SendOTP`**: Generates a One-Time Password, stores it temporarily with a time-to-live, and dispatches it via email or SMS.
- **`VerifyOTP`**: Validates the provided OTP against the temporary store and flags the contact method as verified.
- **`SendMagicLink`**: Generates a secure, cryptographically signed Magic Link for passwordless authentication or verification.
- **`VerifyMagicLink`**: Consumes the Magic Link token securely.
- **`GetWhatsAppVerificationLink`**: Specific flow to generate a verification sequence over WhatsApp.

## 8. Cross-Cutting Security Mechanics (Implicit Features)
_These features operate under the hood across various endpoints listed above._
- **Rate Limiting & Brute Force Protection**: Tracks failed `Login` and `ChallengeMFA` attempts in Redis, locking the account after sequential failures.
- **Tenant Isolation & IP Controls**: Rejects authentication requests across the system if the request IP does not conform to the Tenant's explicitly configured Allowlist or Denylist.
- **Audit Logging**: Asynchronously records critical security events (logins, password changes, suspensions) without blocking the primary request execution.
- **Observability**: Automatically instruments endpoints with OpenTelemetry distributed tracing and Prometheus metrics.
